MWA JPEG Component Library

 This folder contains the complete source code for the MWA JPEG Component Library and build scripts.

The file build.bat should be used to build the component library from source. The first few lines of this script are configurable and specify the paths to the versions of Delphi and C++Builder installed on your system. By default, they are set to typical installation paths. However, you may need to edit them if your installation differs from this. You may ignore the paths to versions of Delphi and C++Builder not installed on your system.

Once Build.bat has completed, you should have two new folders in the source code folder. "Packages" contains the Delphi and C++Builder design and runtime packages for the JPEG Component Library while, the "units" folder contains the .dcu and for C++Builder, the .obj and .hpp files. In each folder, a sub-folder will have been created for each version of Delphi and C++Builder installed on your system and the compiled files placed there.

Installation of Old Versions

The installation script will only automatically install Delphi 6 and later or C++Builder 5 and later. Earlier versions have to be built using the "build.bat" script and manually installed as described below.

Delphi 1 i. Copy the mwjpeg.dll in the dll folder to your windows directory.

ii. Create a new folder in your delphi folder (the actual name is unimportant - call it jpeg if you can't think of anything better) and copy the files contained in the units\d1 folder into this new folder. Y

iii. Start the Delphi IDE and choose the Options|Install Components menu option. Click on the "Add" button and browse for the jpegreg1.pas file you just copied to the new folder. Click on OK. (Note that this file contains the registration information for the JPEG components. By default the two non-visual components are placed on the "Additional" tab, and TDBJPEGImage is on the Data Controls tab. You can edit this file to locate them elsewhere).
Delphi 2 i. Copy the mwjpeg32.dll in the dll folder to your windows directory.

ii. Create a new folder in your delphi folder (the actual name is unimportant - call it jpeg if you can't think of anything better) and copy the files contained in the units\d2 folder into this new folder.

iii. Start the Delphi IDE and choose the Component|Install menu option. Click on the "Add" button and browse for the jpegreg1.pas file (or jpegreg2.pas if you are using Quick Reports 2) you just copied to the new folder. Click on OK. (Note that this file contains the registration information for the JPEG components. By default these are placed on the "Additional" tab, TDBJPEGImage is on the Data Controls tab, and TQRDBJPEGImage is on the QReports tab. You can edit this file to locate them elsewhere).
Delphi 3 i. Create a new folder in your delphi directory (the actual name is unimportant - call it jpeg if you can't think of anything better) and copy the files contained in the units\d3 into this new folder. You should also copy the dclmwajpg30.dpl from the Packages folder into the new folder

iii. Start the Delphi IDE and choose the Component|Install Packages menu item. Click on the "Add" button and browse for the dclmwajpg30.dpl file you just copied to the new subdirectory. Click on OK. The package will now be installed. By default the JPEG components are placed on the "Additional" tab, TDBJPEGImage is on the Data Controls tab, and TQRDBJPEGImage is on the QReports tab.

iv. Now select the Tools|Environment Options menu item, and click on the "Library" tab. You must now add to the end of the "Library Path" the full pathname of the folder that contains the JPEG Components, if it's not already present. Remember to put a semi-colon (;) separator in before you add the path name. For example, if you installed the JPEG Components into "d:\Delphi 3\jpeg", then the Library Path should now look something like:

D:\Delphi 3\Lib;D:\Delphi 3\Bin;D:\Delphi 3\Imports;d:\delphi 3\jpeg

IMPORTANT: You should now restart Delphi before trying to load a JPEG Image at Design Time, and this includes the example applications. If you do not then it is likely that you will get an Access Violation in VCL30.dpl.

Remember to put a semi-colon (;) separator in before you add the path name. C++Builder i. Create a new folder in your C++Builder directory called "jpeg" and copy the files contained in theunits\cb1 folder into this new folder.

ii. Start the C++Builder IDE and then choose the Component|Install menu option. Click on the "Add" button and browse for the jpegreg1.cpp (or jpegreg2.cpp if you are using Quick Reports 2) file you just copied to the new folder. Click on OK. (Note that this file contains the registration information for the JPEG components. By default these are placed on the "Additional" tab, TDBJPEGImage is on the Data Controls tab, and TQRDBJPEGImage is on the QReports tab. You can edit this file to locate them elsewhere).

The components are now installed. Use the demo application to learn how they are used. Note that if you have registered for and installed Quick Reports 2 and have placed the Quick Reports .hpp and .obj files in directories other than the standard include and lib directories, then you may have to modify the demo programs' .mak files so that these directories are included in the search path.
C++Builder 3 i. Create a new folder in your C++Builder directory called "jpeg" and copy the remaining files contained in the units\cb3 folder into this new folder. You should also copy the dclmwajpgcb3.bpl from the Packages folder into the new folder

ii. Start the C++Builder IDE and choose the Component|Install Packages menu item. Click on the "Add" button and browse for the dclmwajpgcb3.bpl file you just copied to the new folder. Click on OK. The package will now be installed. By default the JPEG components are placed on the "Additional" tab, TDBJPEGImage is on the Data Controls tab, and TQRDBJPEGImage is on the QReports tab.

i11. You should now add ";$(BCB)\jpeg" to the end of the library path in the Tools|Environment Options dialog box (it's on the Library tab). This is so that the jpeg components can be automatically found in any project of yours that uses them.

You should now restart C++Builder before trying to load a JPEG Image at Design Time.
